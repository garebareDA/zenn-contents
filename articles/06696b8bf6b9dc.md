# Rubyのframeとは
Ruby VM においてFrame（フレーム、制御フレーム）とは、メソッド呼び出しやブロック実行時に VM スタック上に積まれる制御情報の単位です。
C 言語の構造体 `rb_control_frame_t`（通称 cfp）として表現され、プログラムカウンタ（pc）、スタックポインタ（sp）、命令シーケンス（iseq）、self、block_code などの実行に必要な情報を含んでいます。
Frame はメソッド呼び出し・ブロック・クラス/モジュール定義などを行うたびに積まれ、終了するとポップされます。

```c
typedef struct rb_control_frame_struct {
    const VALUE *pc;        // cfp[0]
    VALUE *sp;              // cfp[1]
    const rb_iseq_t *_iseq; // cfp[2] -- use rb_cfp_iseq(cfp) to read
    VALUE self;             // cfp[3] / block[0]
    const VALUE *ep;        // cfp[4] / block[1]
    const void *block_code; // cfp[5] / block[2] -- iseq, ifunc, or forwarded block handler
    void *jit_return;       // cfp[6] -- return address for JIT code
#if VM_DEBUG_BP_CHECK
    VALUE *bp_check;        // cfp[7]
#endif
} rb_control_frame_t;
```

| フィールド | 役割 |
|-----------|------|
| `pc` | プログラムカウンタ（現在実行中の命令アドレス） |
| `sp` | スタックポインタ（VM スタックの先頭） |
| `iseq` | 実行中の命令シーケンス |
| `self` | 現在のレシーバ |
| `block_code` | ブロック実行時のコードポインタ |
| `jit_return` | JIT コンパイルされたコードからの戻り先 |
| `ep` | 環境ポインタ（ローカル変数へのアクセスに使用） |
| `__bp__` | ベースポインタ |

# Lightweight Frames（ZJIT）
Lightweight FramesはZJIT向けに導入された最適化です。
- PR: [#16262](https://github.com/ruby/ruby/pull/16262) "ZJIT: Lightweight Frames
- マージ済み

- メリット: JIT-to-JIT 呼び出しのオーバーヘッド削減、ホットパスの高速化
- デメリット: コードサイズ増加、exit パスの複雑化、メモリ使用量増加

## 仕組み

従来の JIT コンパイルでは、JIT-to-JIT のメソッド呼び出しにおいても毎回以下のフィールドを VM スタックに書き込む必要がありました。
- `cfp->pc` （プログラムカウンタ）
- `cfp->iseq` （命令シーケンス）
- `cfp->block_code` （ブロックコード）

Lightweight Frames では、ZJIT がコンパイル時に専用の JITFrame を割り当て、そのポインタを `cfp->jit_return` に書き込むことで、上記 3 つのフィールドの書き込みを最適化を行っています。
