# TypeScriptの型演算子とは

TypeScriptの型システムでは、型を組み合わせたり変換したりするための演算子が用意されています。

```ts
// ユニオン型 - いずれかの型
type A = string | number;

// インターセクション型 - すべてを満たす型
type B = Foo & Bar;

// 型演算子 - 型を修飾する演算子
type C = keyof T;
type D = typeof obj;
type E = readonly T[];
type F = T[K];     // Indexed Access

// 配列型
type G = T[];
```

# 型の演算子の優先順位

| 優先順位 | 構文 | パース関数 |
|---------|------|-----------|
| 高い | Array（`T[]`）、Indexed Access（`T[K]`） | `parsePostfixTypeOrHigher()` |
| ↓ | TypeReference（`Foo<T>`） | `parseNonArrayType()` |
| ↓ | Type Operators | `parseTypeOperatorOrHigher()` |
| ↓ | Intersection | `parseIntersectionTypeOrHigher()` |
| 低い | Union | `parseUnionTypeOrHigher()` |

```ts
type A = keyof T & U[];  // (keyof T) & (U[])
type B = keyof (T & U);  // 括弧がなければ (keyof T) & U
type C = A | B & C;      // A | (B & C)
```

# どうやって決まっているか

ECMAScript には仕様書に演算子優先順位表が掲載されていますが、TypeScript の型システムには最新実装と同期した正式な仕様書や優先順位表はありません。

型の解析は `src/compiler/parser.ts` の再帰下降パーサによって行われています。再帰下降パーサでは、構文の入れ子関係を 関数の呼び出し階層で表現します。

```ts
// エントリポイント
function parseType(): TypeNode {
    const type = parseUnionTypeOrHigher();
    // 条件型の処理...
    return type;
}
```

`parseType()` の中では、以下の順に関数が呼び出されます。

```
parseType()
    ↓
parseUnionTypeOrHigher()      // |
    ↓
parseIntersectionTypeOrHigher() // &
    ↓
parseTypeOperatorOrHigher()   // keyof typeof readonly unique infer
    ↓
parsePostfixTypeOrHigher()    // [] [K]
```

この `parse○○OrHigher()` の責務分割が優先順位を表現しています。

上位の関数（`parseUnionTypeOrHigher`）が最も弱く結合し、下位の関数（`parsePostfixTypeOrHigher`）が最も強く結合します。

コードを抜粋します（※説明のため簡略化しています）。

```ts
function parseUnionTypeOrHigher(): TypeNode {
    return parseUnionOrIntersectionType(SyntaxKind.BarToken, parseIntersectionTypeOrHigher, ...);
}

function parseIntersectionTypeOrHigher(): TypeNode {
    return parseUnionOrIntersectionType(SyntaxKind.AmpersandToken, parseTypeOperatorOrHigher, ...);
}

function parseTypeOperatorOrHigher(): TypeNode {
    if (token()はkeyof/readonly/unique/infer) return その処理;
    return parsePostfixTypeOrHigher();
}

function parsePostfixTypeOrHigher(): TypeNode {
    let type = parseNonArrayType();
    while (token() === SyntaxKind.OpenBracketToken) {
        // C[] または C[K] を処理
    }
    return type;
}
```

## 処理を追ってみる

`A | B & C[]` のパースを追ってみます。

1. `parseUnionTypeOrHigher` が `|` を発見 → 左右を `parseIntersectionTypeOrHigher` に要求
2. 右辺 `B & C[]` → `parseIntersectionTypeOrHigher` が `&` を発見
   - `parseTypeOperatorOrHigher` → `parsePostfixTypeOrHigher` が `[]` を拾い `C[]`
3. 結果: `A | (B & (C[]))`

# まとめ
TypeScript の型演算子の優先順位は、仕様書の優先順位表ではなく、コンパイラの再帰下降パーサの構造によって表現されています。
この方式は演算子が増えると関数の階層やコードの重複が増えやすいという側面があります。一方で、TypeScript の型演算子は種類が比較的少ないため、シンプルで理解しやすい実装になっています。
